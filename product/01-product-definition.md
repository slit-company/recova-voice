# 제품 정의

- 문서 상태: active
- 마지막 검토: 2026-07-10

## 제품

Recova는 한국 B2B 팀을 위한 셀프서브 AI 전화 에이전트 빌더다.

사용자는 대화 워크플로를 만들고, 웹 또는 실제 전화로 시험하고, 인바운드·아웃바운드·캠페인에 적용한 뒤, 전사·도구 호출·통화 결과·비용을 보고 다시 개선할 수 있어야 한다.

## 해결하려는 문제

한국의 영업·고객지원·콜센터 팀이 AI 전화를 도입하려면 다음을 여러 제품과 개발 인력으로 이어 붙여야 한다.

- 업무 대화와 분기 설계
- 한국어 음성 모델과 전화 연결
- 단건 및 대량 발신, 인바운드 수신
- 녹취·전사·도구 호출·실패 원인 확인
- 조직별 권한, 사용량, 비용, 감사
- 실제 업무 성과 측정과 반복 개선

Recova는 이 과정을 하나의 운영 가능한 제품 흐름으로 묶는다.

## 핵심 사용자

| 사용자 | 필요한 결과 |
|---|---|
| 현업 운영자 | 개발자 없이 에이전트를 만들고 시험하며 반복 수정한다. |
| 영업·지원·콜센터 관리자 | 캠페인과 인바운드 업무를 운영하고 결과와 실패를 관리한다. |
| 회사 도입 결정자 | 실제 통화 품질, 업무 성과, 비용, 보안과 운영 가능성을 확인한다. |
| 조직 관리자 | 사용자·API 키·전화 설정·사용량·권한을 조직 단위로 통제한다. |
| Recova superadmin | 조직과 전화번호 inventory를 tenant-safe하게 운영하고 장애를 추적한다. |

첫 시장 세부 업종과 구매자 우선순위는 아직 확정되지 않았다. 확인되지 않은 업종 특화 요구를 제품 전체 기본값으로 만들지 않는다.

## 제품 약속

1. **빠른 첫 가치:** 가입한 사용자가 최소 입력으로 에이전트를 만들고 첫 의미 있는 통화를 완료한다.
2. **빌더 중심:** 전화번호나 공급자 설정이 아니라 업무 대화와 결과 설계가 사용자의 중심 작업이다.
3. **시험에서 운영까지:** 테스트 성공이 인바운드, 단건 아웃바운드 또는 캠페인 운영으로 자연스럽게 이어진다.
4. **측정 가능한 결과:** 전사, 녹취, 도구 호출, disposition, 실패 원인, 지연과 비용을 확인할 수 있다.
5. **반복 개선:** 결과 화면에서 문제를 이해하고 해당 에이전트를 수정·재시험할 수 있다.
6. **한국 B2B 운영성:** 한국어 통화, 국내 전화번호, 조직 관리, 대량 운영과 감사 요구를 우선한다.

## 제품 원칙

- 셀프서브 데모는 일회성 쇼가 아니라 회사 도입 평가 흐름이다.
- 기본 데모 통화는 표준 STT + LLM + TTS 파이프라인을 사용한다.
- Realtime은 선택 가능한 저지연 경로지만, 한국 B2B 실통화 근거가 기준을 충족하기 전에는 기본값이 아니다.
- 안정적인 전사·리포트·도구 추적·비용 가시성·장애 분석을 최저 지연보다 우선한다.
- 캠페인, 리포트, 사용량, 녹음, 전화 설정, API 키, 조직과 superadmin은 데모 장식이 아니라 production 표면이다.
- 모든 조직 범위 읽기·쓰기는 `organization_id`를 검증한다. tenant isolation은 타협할 수 없다.
- 실제 도메인·이미지·링크·패키지가 준비되지 않은 Dograh 표면은 맹목적으로 Recova로 바꾸지 않는다.
- simulator나 synthetic benchmark를 실제 PSTN 품질 또는 production readiness로 표현하지 않는다.

저장소 작업 규칙은 [`../AGENTS.md`](../AGENTS.md), 런타임별 기술 조건은 [`../docs/product/realtime-development-direction.md`](../docs/product/realtime-development-direction.md)와 [`../docs/product/returnzero-latency-optimization.md`](../docs/product/returnzero-latency-optimization.md)에서 관리한다. 이 문서는 그 세부 구현 상태를 복제하지 않는다.

## 제품 범위

### 포함

- 워크플로 기반 음성 에이전트 생성과 편집
- 웹콜, 전화 프리뷰, 텍스트 시뮬레이션을 통한 시험
- 인바운드와 아웃바운드 전화 운영
- 캠페인, 도구, 파일, 녹음, 모델과 전화 설정
- 실행 상세, 전사, 리포트, 사용량과 비용
- 조직, 권한, API 키와 superadmin 운영
- 한국어 UX와 국내 전화 인프라 연동

### 현재 핵심 범위가 아님

- 일반 소비자용 개인 비서
- 전화기나 Centrex 단말 판매
- 고객에게 raw carrier/Jambonz credential을 노출하는 BYOC 관리 화면
- 근거 없는 전면 Recova 리브랜딩
- Realtime을 무조건 기본값으로 만드는 것
- 공급자 계약 전 특정 SIP header·CDR·실패 코드 추측 구현

## 현재 상태 원본

제품 역량과 검증 상태의 단일 원본은 [`03-capability-map.md`](./03-capability-map.md)다. 자체 070의 외부 공급자 대기는 빌더·온보딩·시험→운영→측정 개선을 막지 않으며, 대기 이유와 기술 handoff는 `context/`에서 관리한다.

## 열린 제품 결정

- 첫 우선 업종과 업무: 영업, 고객지원, 콜센터 중 어디를 첫 wedge로 삼을지
- 첫 구매자와 챔피언: 현업 운영자, 팀장, 개발/IT, 경영진 중 누구인지
- SaaS와 self-hosted 중 초기 판매의 기본 모델
- 무료 시험 범위, OTP, 크레딧, 과금 전환 조건
- 인바운드 체험을 계약 전 사용자에게 어떤 방식으로 제공할지
- 녹음 동의, 보존·삭제, 개인정보와 업종별 규제 정책
- 회사 도입 평가를 통과했다고 볼 핵심 비즈니스 결과
